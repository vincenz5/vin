Blockchain Knowledge

A blockchain is based on a set of rules that enforce an accounting system accounting for balances, like on Bitcoin, plus states, like on Ethereum. The system of rules are designed in a method that follows game theory (actually mechanism design but that's not common knowledge).

The rules set forth by the protocol of a blockchain are intended to both coordinate the verifyers of new sets of data (miners) but also pit them up against each other so they cannot make up data. If they could make up data, they could increase the balance of their own wallets or decrease those of others.

What this specifically solves is problems like double spending and lack of transparency. Since transactions cannot be falsified and the blockchain ledger (the data showing all transactions and balances, plus contract code like in virtual machines) is shared among anyone with a running node, it makes for a form of governance of accounting that anyone can rely on without trusting the intentions or abilities of third parties. This compares to the existing methods of the financial system where balances and accounting of the monetary system and debts are held on private ledgers, on corporate and government servers, and transfers are based on faith of the institutionns involved.

Bitcoin really has two functions:

  • The first, and certainly best know: accounting of balances of the native currency, Bitcoin, in people's wallets. Wallets are tracked through their wallet addresses, hexidecimal (0 through 9 and a through f) lines (public keys) 26 to 35 characters long. These addresses are controlled by whoever (or whatever) inputs the private key (64 characters long). Something like: E9873D79C6D87DC0FB6A5778633389F4453213303DA61F20BD67FC233AA33262

  • The much less known: simple escrow between public addresses.

Ethereum introduces new functions:

  • Smart contracts that can be encoded through the use of Solidity, and compiled into bytecode to run on the Ethereum Virtual Machine that is verified by Ethereum miners. These contracts are self enforcing, which means they can follow their own encoded commands based on the predetermined rules programmed into them by their authors.

  • Escrow contracts that can include functions like time or conditional lockup.

  • Roles and permissions can be set in immutable code on these contracts, allowing for reduction in corruption and improved tools for transparency of governance.

  • Tradable or non tradable tokens that are transferable between addresses with the permission of the governing smart contract.

  • State channels: contracts that govern the transaction between at least two addresses. These can be used to govern payments or other forms of state. These channel contracts can be encoded to accept off-chain messages that are signed by the private keys of the addresses involved in the transaction and enforcement of payments, for example, can be enforced through collateral locked up by the smart contract and only released according to the rules encoded in the contract. This allows for multiple parties to interact without having the iniate and wait for on-chain transactions every time while still relying on self enforcement of smart contracts and the immutability of a blockchain.

  • Coordination of voting and exection of governance mandates set by decentralized organizations of people and entities.

  • Decentralization of exchanges and methods of price discovery. For example, one can set up a blind auction model where players can read the Solidity code regulating the auction but cannot see what everyone else is offering during the auction. Also, a contract exchange can be set up to hold tokens or Ether in escrow during trades.

  • Trustless operation of opening and closing of sidechains, where parties can enter and leave by staking something in a contract on the mainchain to enter into the sidechain but verification of transactions and other changes of state or storage on the sidechain is verified according to the specific rules of that sidechain. For example, xDai is a sidechain where Dai tokens from the mainchain can be employed on the sidechain, with a proof of authority protocol (reduced security) but almost instant transactions at almost no cost.

Javascript

A language for controlling specific parts of a website.
Accepted by the user's browser natively.

Allows the developer of a website to do things like:

  • Edit specific parts of the website.
  • Have the website react to actions taken by the website visitor or other specified events.
  • Connect the website over to the backend server software of the website or even a completely different entity. (APIs)

Typescript

A language that is converted into plain Javascript once it is on the browser of the website visitor. (Transpiled)

Differences from Javascript:

  • Extension of file is .ts instead of .js
  • It's sort of like SCSS for CSS. It's just a developer tool to build more robust Javascript and having an easier time catching errors.
  • It makes it easier for the programmer to catch errors before running, while typing the code.
  • Still accepts Javascript libraries
  • Typescript infers types for variables that aren't explicitally written by the developer.
  • It is an object oriented language, making it more like backend languages like Python or other languages that allow for automation of processes.
  • Javascript, instead, is a scripting language, which means it is a program controlling another program. Javascript on the front end is controlling the web browser being used to view the website.
  • Scripting languages are interpreted down to something like bytecode, not compiled all the way down to machine code. This is what makes it 'higher level'. The web browser program is 'lower level', meaning it it compiles down into machine code.
  • It is a statically typed language, which means the program cannot work around the type system of the language and the program is checked before execution.
  • The program may be rejected before even starting but it won't run outside the language type.
  • The opposite is dynicamically type language, which just runs the program even if there will be errors.
  • In statically typed language, the type is bound to the variable while dynicamically typed languages bound the type to the value.

    • As an example:

  • string a = "blah"
  • In Typescript, a will always remain a string. The value of s ("blah") may change during its life but it will never change to an integer or a boolean, for example.
  • If Typescript is not compatible with the browser because of its ES6 features, it will compile down into ES3, ES4, or ES5 for compatibility.
  • Typescript does not work with abstract classes.

Daily Meditiation Techniques Following Marcus Aurelius's Medititations

1. Contemplation of the Sage

a. Look up to the character traits of either:

i. Someone you look up to, specifically looking up to their character traits

ii. Who you want yourself to be, specifically what are the character traits of the person you want yourself to be

2. Look from above

a. Zoom out from yourself into third person

b. Continue to zoom out even further

c. Zoom out to a worldwide third person view of what's around you

d. Build out perspective of yourself based on everything else

e. Disassociate yourself from your emotions

f. Focus on this when making decisions

3. Exercise

a. Not for physical benefits but for mental benefits

4. Meditation

a. Focus on not reacting to your emotions without thought

5. Review of the day

a. Look back on the day and on your actions

b. Congratulate yourself for the good things you did to reinforce them

c. Identify the problems and where you went wrong

d. Focus on not doing the problematic actions the next day

e. Eliminate the rationalizations you give yourself that lead to the problematic actions

f. Check on whether you're in track to #1 and what you need to do different

6. Journaling

a. Make it into a habit

b. Define the problem you're having and consider it objectively

Effects:

Feel more happy and fulfilled because you focus on the things you need to do and you truly are fulfilled by. Your accomplishments will make you more confident.
